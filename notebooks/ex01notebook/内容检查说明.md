# 内容检查说明

## 1. 内容检查的目的

文件级检查只能确认 audio 文件和 MIDI 文件是否存在，但不能说明这些文件是否真的可用于实验。内容检查的目标是进一步验证文件内部是否正常，包括音频是否可读取、MIDI 是否可读取、MIDI 中是否有音符、音频和 MIDI 的时长是否基本一致。

本阶段的核心问题是：

| 问题 | 含义 |
|---|---|
| 音频能否读取 | `.wav` 文件是否可以被 Python 正常打开 |
| MIDI 能否读取 | `.mid` / `.midi` 文件是否可以被正常解析 |
| 音频是否正常 | 是否为空、静音、削波、采样率异常 |
| MIDI 是否有效 | 是否包含 note event，音高范围是否合理 |
| audio-MIDI 是否对齐 | 音频时长和 MIDI 标注时长是否接近 |
| 样本是否可用 | 是否可以进入后续 AMT 推理、训练或评估 |

## 2. Audio 内容检查

Audio 内容检查用于判断音频文件本身是否正常。

| 字段名 | 中文含义 | 检查目的 |
|---|---|---|
| `audio_path` | 音频路径 | 记录音频文件的位置 |
| `audio_readable` | 音频是否可读取 | 判断音频文件是否损坏或格式不支持 |
| `sample_rate` | 采样率 | 检查音频采样率是否统一，例如 44100 Hz |
| `num_channels` | 声道数 | 判断音频是单声道还是双声道 |
| `num_samples` | 采样点数量 | 用于计算音频长度 |
| `audio_duration_sec` | 音频时长，单位为秒 | 判断音频是否过短或过长 |
| `max_amplitude` | 最大振幅 | 判断音频是否存在削波或异常峰值 |
| `rms_amplitude` | 均方根能量 | 判断音频整体音量是否过低 |
| `is_silent` | 是否疑似静音 | 过滤全静音或近似静音音频 |
| `is_clipped` | 是否疑似削波 | 判断音频是否有大量振幅接近最大值 |

## 3. MIDI 内容检查

MIDI 内容检查用于判断标注文件是否有效。

| 字段名 | 中文含义 | 检查目的 |
|---|---|---|
| `midi_path` | MIDI 路径 | 记录 MIDI 文件的位置 |
| `midi_readable` | MIDI 是否可读取 | 判断 MIDI 文件是否损坏或格式异常 |
| `num_notes` | 音符数量 | 判断 MIDI 中是否包含有效音符 |
| `midi_duration_sec` | MIDI 时长，单位为秒 | 用于和音频时长进行对比 |
| `min_pitch` | 最低音高 | 检查音高范围是否异常 |
| `max_pitch` | 最高音高 | 检查音高范围是否异常 |
| `mean_pitch` | 平均音高 | 辅助观察 MIDI 音高分布 |
| `num_instruments` | 乐器轨道数量 | 判断 MIDI 结构复杂度 |
| `num_programs` | MIDI program 数量 | 观察 MIDI 中是否包含多个音色设置 |

---

## 4. Audio-MIDI 对齐检查

Audio-MIDI 对齐检查用于判断音频和 MIDI 标注是否属于同一个样本，并且时长是否基本一致。

| 字段名 | 中文含义 | 检查目的 |
|---|---|---|
| `audio_duration_sec` | 音频时长 | 作为真实音频长度参考 |
| `midi_duration_sec` | MIDI 时长 | 作为标注长度参考 |
| `duration_diff_sec` | 音频和 MIDI 的时长差 | 判断二者是否明显不匹配 |
| `duration_match` | 时长是否匹配 | 若时长差小于阈值，则认为基本匹配 |

本实验初步使用较宽松的判断标准：

| 判断条件 | 含义 |
|---|---|
| `duration_diff_sec <= 1.0` | 认为 audio 和 MIDI 时长基本匹配 |
| `duration_diff_sec > 1.0` | 标记为时长不匹配，需要人工复查 |

时长不匹配可能来自以下原因：

| 问题类型 | 可能原因 |
|---|---|
| 音频和 MIDI 文件配错 | 文件名或 metadata 对应关系错误 |
| MIDI tempo 异常 | MIDI 解析时节拍信息处理错误 |
| 音频包含额外静音 | 音频开头或结尾有较长空白 |
| MIDI 标注不完整 | MIDI 只覆盖音频的一部分 |
| 合成流程问题 | 由 MIDI 渲染音频时出现参数错误 |

## 5. 样本有效性判断

最终需要给每个样本生成一个 `is_valid` 字段，用来判断该样本是否可以进入后续实验。

| 条件 | 要求 |
|---|---|
| `has_audio` | 必须为 `True` |
| `has_midi` | 必须为 `True` |
| `audio_readable` | 必须为 `True` |
| `midi_readable` | 必须为 `True` |
| `is_silent` | 必须为 `False` |
| `num_notes` | 必须大于 0 |
| `duration_match` | 建议为 `True` |

若以上条件全部满足，则：

| 字段 | 取值 |
|---|---|
| `is_valid` | `True` |
| `problem_type` | `valid` |

如果存在问题，则：

| 字段 | 取值 |
|---|---|
| `is_valid` | `False` |
| `problem_type` | 对应异常类型 |

## 6. 异常类型说明

| `problem_type` | 中文含义 | 说明 |
|---|---|---|
| `valid` | 正常样本 | 可以进入后续实验 |
| `audio_not_readable` | 音频无法读取 | 音频文件损坏或格式不支持 |

## 结果
## organ duration mismatch 处理说明

初始检查中使用 `duration_diff_sec <= 1.0` 作为 audio-MIDI 时长匹配阈值，发现 15 条 organ 样本被标记为 `duration_mismatch`。

进一步检查发现，这些样本的时长差均集中在约 2.4–2.5 秒，并且全部表现为 `audio_duration_sec > midi_duration_sec`。由于差值方向一致且数值稳定，判断该现象更可能来自 MIDI 合成音频时保留的尾音或结尾静音，而不是文件配对错误。

因此后续将时长阈值放宽为：

`duration_diff_sec <= 3.0`
